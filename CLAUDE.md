# TGGeo - Telegram GeoGuessr Mini App

## Описание проекта
Мини-игра в стиле GeoGuessr, интегрированная в Telegram через WebApp. Пользователи видят случайное фото из Mapillary и должны угадать местоположение на карте.

## Архитектура (обновлено 2026-05-05)

### Современная serverless архитектура
Проект полностью переработан для работы **без Flask и ngrok**:

1. **Статический WebApp** (`index.html`) - размещается на GitHub Pages/Vercel/Netlify
2. **Telegram бот** (`bot.py`) - работает локально или на любом сервере
3. **Прямая передача данных** - через Telegram WebApp API, без промежуточного сервера

### Технологический стек
- **Backend**: aiogram 3.x (async Telegram Bot API)
- **Frontend**: Vanilla JS + Leaflet.js (карты) + Telegram WebApp API
- **API**: Mapillary (фотографии улиц)
- **Геокалькуляции**: geopy
- **Конфигурация**: python-dotenv

## Основные функции

### bot.py
- `start()` - обработчик команды `/start`, показывает кнопку для запуска игры
- `handle_webapp_data()` - принимает результат от WebApp, вычисляет расстояние и очки
- `handle_unknown()` - обработчик неизвестных команд
- Полное логирование всех действий
- Валидация входящих данных
- Обработка всех типов ошибок

### index.html
- Загружает случайное фото из Mapillary API (клиентская сторона)
- Интерактивная карта Leaflet для выбора точки
- Отправляет результат в бот через Telegram WebApp API
- Обработка ошибок загрузки
- Адаптивный дизайн
- Поддержка темной темы Telegram

## Игровая механика
1. Пользователь нажимает "Играть" в боте
2. Открывается WebApp с фото и картой мира
3. Пользователь кликает на карту, выбирая предполагаемую локацию
4. Нажимает "Угадать"
5. Бот вычисляет расстояние и начисляет очки: `max(0, 5000 - distance_km)`

## Рекомендации по развертыванию

### Локальная разработка
1. Установить зависимости: `pip install -r requirements.txt`
2. Скопировать `.env.example` в `.env` и заполнить токены
3. Временно использовать ngrok для тестирования: `ngrok http 8000` и запустить `python -m http.server 8000`
4. Запустить бота: `python bot.py`

### Production (рекомендуется)
1. **WebApp**: Разместить `index.html` на GitHub Pages/Vercel/Netlify (бесплатно, HTTPS автоматически)
2. **Бот**: Запустить на любом сервере (VPS, Raspberry Pi, даже домашний ПК)
3. **Конфигурация**: Обновить `WEBAPP_URL` в `.env` на production URL
4. **Мониторинг**: Использовать systemd/docker/pm2 для автозапуска

### Варианты хостинга WebApp
- **GitHub Pages**: бесплатно, просто, `https://username.github.io/tggeo`
- **Vercel**: бесплатно, быстро, автодеплой из Git
- **Netlify**: бесплатно, drag-and-drop интерфейс
- **Cloudflare Pages**: бесплатно, CDN по всему миру

## Технические детали

### Почему без Flask?
Старая архитектура требовала:
- Запущенный Flask сервер 24/7
- ngrok для туннелирования (нестабильный URL)
- Дополнительные зависимости
- Сложное развертывание

Новая архитектура:
- Статический HTML на CDN (быстро, надежно)
- Бот работает независимо
- Простое развертывание
- Бесплатный хостинг

### Как работает передача данных?
1. Пользователь открывает WebApp
2. JavaScript загружает фото из Mapillary API
3. Пользователь выбирает точку на карте
4. `tg.sendData()` отправляет JSON напрямую боту
5. Бот получает данные через `web_app_data` handler
6. Вычисляет расстояние и отправляет результат

### Лимиты Mapillary API
- Бесплатный tier: 50,000 запросов/месяц
- Для проекта достаточно (1-2 запроса на игру)
- При превышении можно добавить кеширование или несколько токенов

## Контакты и поддержка
При работе с этим проектом учитывайте:
- Mapillary API имеет лимиты запросов (50k/месяц бесплатно)
- Telegram WebApp требует HTTPS (GitHub Pages/Vercel предоставляют автоматически)
- Бот может работать на любом устройстве с Python 3.8+

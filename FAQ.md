# ❓ Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Сколько это стоит?
**Бесплатно!** 
- GitHub Pages - бесплатно
- Mapillary API - 50,000 запросов/месяц бесплатно
- Telegram Bot API - бесплатно
- Бот можно запускать на своем компьютере

Платить нужно только если хотите сервер 24/7 (от $3-5/месяц), но есть и бесплатные варианты.

### Нужно ли мне знать программирование?
**Нет!** Просто следуйте инструкциям шаг за шагом. Вам нужно:
1. Скопировать несколько команд в терминал
2. Создать файл `.env` и вставить туда токены
3. Загрузить один файл на GitHub

### Будет ли бот работать если я выключу компьютер?
**Нет.** Бот работает только пока запущен `bot.py`. Варианты:
- Оставить компьютер включенным
- Запустить на сервере (VPS)
- Использовать Raspberry Pi или старый ноутбук

### Могу ли я изменить города в игре?
**Да!** Откройте `index.html`, найдите массив `cities` (строка 18) и добавьте свои координаты:
```javascript
const cities = [
    {name: "Ваш город", lat: 55.7558, lon: 37.6176},
    // ... добавьте больше
];
```

### Как добавить больше языков?
Сейчас бот на русском. Чтобы добавить другие языки:
1. В `bot.py` измените тексты сообщений
2. В `index.html` измените тексты кнопок и подсказок

## Технические вопросы

### Почему GitHub Pages, а не свой сервер?
- **Бесплатно** - не нужно платить за хостинг
- **Быстро** - CDN по всему миру
- **Надежно** - 99.9% uptime
- **Просто** - загрузил файл и готово
- **HTTPS** - автоматически (требование Telegram)

### Можно ли использовать другой хостинг вместо GitHub Pages?
**Да!** Любой хостинг с HTTPS подойдет:
- Vercel
- Netlify
- Cloudflare Pages
- GitLab Pages
- Даже свой сервер с Nginx

Главное - HTTPS и доступность `index.html` по URL.

### Зачем нужен Mapillary токен?
Mapillary - это база фотографий улиц (как Google Street View). Токен нужен для доступа к API. Бесплатный лимит - 50,000 запросов/месяц, этого хватит на тысячи игр.

### Можно ли использовать другой источник фотографий?
**Да!** Можно заменить Mapillary на:
- Google Street View API (платный)
- Свою коллекцию фотографий
- Любой другой API с геолокацией

Нужно изменить функцию `fetchImage()` в `index.html`.

### Сколько пользователей может играть одновременно?
**Неограниченно!** 
- `index.html` на GitHub Pages выдерживает миллионы запросов
- `bot.py` обрабатывает сотни пользователей одновременно
- Ограничение только по Mapillary API (50k запросов/месяц)

### Как добавить базу данных для статистики?
Нужно:
1. Установить SQLite или PostgreSQL
2. Создать таблицу для хранения игр
3. В `bot.py` добавить сохранение результатов
4. Создать команду `/stats` для просмотра статистики

Это требует знаний Python и SQL.

### Можно ли добавить лидерборд?
**Да!** Нужна база данных (см. выше) и:
1. Сохранять `user_id` и очки каждой игры
2. Создать команду `/leaderboard`
3. Выводить топ-10 игроков

### Как защитить токены от кражи?
- ✅ Используйте `.env` файл (уже сделано)
- ✅ Добавьте `.env` в `.gitignore` (уже сделано)
- ✅ Никогда не коммитьте `.env` в Git
- ✅ Не показывайте `.env` на скриншотах
- ✅ Если токен утек - создайте нового бота у @BotFather

### Бот медленно отвечает, как ускорить?
Возможные причины:
1. **Медленный интернет** - проверьте скорость
2. **Далеко от серверов Telegram** - используйте VPS ближе к вам
3. **Mapillary API медленный** - добавьте кеширование фотографий
4. **Много пользователей** - оптимизируйте код или используйте более мощный сервер

### Как обновить бота после изменений?
1. Измените код в `bot.py` или `index.html`
2. Если изменили `index.html` - загрузите на GitHub Pages снова
3. Если изменили `bot.py` - остановите (Ctrl+C) и запустите снова
4. Изменения в `.env` требуют перезапуска бота

### Можно ли запустить несколько ботов?
**Да!** Для каждого бота:
1. Создайте нового бота у @BotFather (новый токен)
2. Скопируйте папку проекта
3. Создайте отдельный `.env` с новым токеном
4. Запустите `bot.py` в каждой папке

### Как мониторить работу бота?
- Смотрите логи: `python bot.py >> bot.log 2>&1`
- Используйте `systemd` на Linux (автоматические логи)
- Добавьте уведомления в Telegram при ошибках
- Используйте мониторинг сервера (Grafana, Prometheus)

## Проблемы и ошибки

### "Address already in use"
Эта ошибка не должна появляться (бот не использует порты). Если появилась - перезапустите бота.

### "Connection timeout"
Проблемы с интернетом или Telegram API недоступен. Подождите и попробуйте снова.

### "Rate limit exceeded"
Превышен лимит Mapillary API (50k/месяц). Варианты:
- Подождите до следующего месяца
- Создайте еще один токен Mapillary
- Добавьте кеширование фотографий

### Бот отвечает, но WebApp показывает белый экран
1. Откройте URL из `WEBAPP_URL` в браузере
2. Нажмите F12 и посмотрите консоль на ошибки
3. Проверьте что `index.html` загружен правильно
4. Проверьте что Mapillary токен в `index.html` правильный

### Как откатить изменения если что-то сломал?
Если используете Git:
```bash
git checkout bot.py  # откатить bot.py
git checkout index.html  # откатить index.html
```

Или просто скачайте файлы заново из репозитория.

# ❓ FAQ - Частые вопросы

## 🚀 Деплой и хостинг

### Где лучше хостить бота?
**Для бесплатного хостинга:**
- **Render.com** (рекомендуется) - засыпает после 15 мин, но решается cron-job
- **Railway.app** - $5 кредитов в месяц
- **Fly.io** - 3 бесплатных VM

**Для платного:**
- **VPS** (DigitalOcean/Hetzner) - $4-6/мес, полный контроль

### Как предотвратить засыпание бота на Render?
Используйте бесплатный сервис [cron-job.org](https://cron-job.org):
1. Создайте аккаунт
2. Добавьте cronjob с URL вашего Render сервиса
3. Интервал: каждые 10 минут
4. Бот будет работать 24/7!

### Нужен ли мне сервер для WebApp?
Нет! `index.html` - это статический файл. Разместите его на:
- GitHub Pages (бесплатно, рекомендуется)
- Vercel (бесплатно)
- Netlify (бесплатно)
- Cloudflare Pages (бесплатно)

## 🔑 Токены и безопасность

### Где взять Telegram Bot Token?
1. Откройте [@BotFather](https://t.me/botfather)
2. Отправьте `/newbot`
3. Следуйте инструкциям
4. Скопируйте токен

### Где взять Mapillary Token?
1. Зарегистрируйтесь на [mapillary.com](https://www.mapillary.com/)
2. Перейдите в [Developer Settings](https://www.mapillary.com/dashboard/developers)
3. Создайте новое приложение
4. Скопируйте **Client Token**

## 🐛 Проблемы и решения

### Бот не отвечает на /start
**Проверьте:**
1. Запущен ли `bot.py`?
2. Правильный ли `TELEGRAM_BOT_TOKEN`?
3. Есть ли ошибки в логах?

### WebApp не открывается
**Проверьте:**
1. GitHub Pages активирован?
2. `WEBAPP_URL` правильный?
3. Прошло ли 2-3 минуты после включения Pages?

### Фото не загружается
**Причины:**
1. Неправильный `MAPILLARY_TOKEN` в `index.html`
2. Токен неактивен
3. Превышен лимит API (50k запросов/месяц)
